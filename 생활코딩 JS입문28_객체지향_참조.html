복제

전자화의 가장 큰 장점은 복제가 쉽다는 것이다.

참조

원시데이터 : 변화를 공유하지 않는다(복제).

쉽게말해, 원시데이터는 copy를 했을 때, 그 순간 둘은 내용은 서로 같지만 다른 값이라 보면 된다.

객체데이터 : 변화를 공유한다(참조).

객체데이터는 참조된다한다
쉽게말해, 객체데이터는 copy를 했을 때, 그 순간 둘은 내용도 같으며, 서로 연동되어있어서 수정되면 같이수정된다. -> 서로 같은 객체를 바라보고있음

flyer할 때 객체를 다른 변수로 copy했을 때 수정을 하면 둘 다 바뀌었음.

함수와 참조

원시데이터타입을 인자로 넘겼을 때

원시데이터타입이기 때문에, 복제가 되어 서로 값을 공유하지 않음.

참조데이터타입을 인자로 넘겼을 때

참조데이터타입이긴 하지만, 함수 내에서 보내진 인자(받은 매개변수)가 다른 객체를 바라보도록 설정하였기 때문에 값은 공유하지 않음.
근데 위처럼 함수 내의 매개변수의 값은 지역변수라 공유안한다고 보는게 맞을듯?

다른예시

참조데이터타입에서, 함수로 보내진 인자(받은 매개변수)가 다른 객체를 바라보는것이 아닌, 그 객체 내의 값을 바꿨기 때문에 공유되고 있음.
